# 32bit 어플리케이션 vs 64bit 어플리케이션
## bit

- binary digit
- 2진수 자리수
- 컴퓨터 처리정보의 최소 단위
- 1바이트 = 8비트

## 64비트, 32비트 차이

- 저장 장치 bit 폭(용량)
    - 32bit 가 한번에 처리할 수 있는 정보 = 2^32
    - 64bit 가 한번에 처리할 수 있는 정보 = 2^64

## 비트가 크면 좋은점

- 한번에 처리할 수 있는 데이터 양이 늘어남 → 처리 고속화
- 64bit CPU가 32bit 명령을 수행할 수 있지만, 그 반대는 안됨
    - 이론적으로는 이렇지만 실제로 64bit 환경에서 32bit 응용프로그램 쓰려면 라이브러리 있어야 한다고 함
    - mac OS Catalina부터는 32bit 응용프로그램 실행 안되니 64bit만 쓰자
- 메모리 제한이 줄어든다 (32bit 환경에서는 메모리 4G까지밖에 안된다)
    - 32bit = 2^32
    - 1GB = 2^30
    - 2^32 / 2^30 = 4 ⇒ 4GB까지밖에 안된다

## 메모리의 계층구조
### 레지스터

- cpu에 존재하는 다목적 저장공간
- 데이터와 명령어를 저장 (CPU가 요청 처리하는데 필요한 데이터 일시적 저장)
- 컴퓨터의 프로세서 내에서 자료를 보관하는 아주 빠른 기억장소
- 현대 프로세서 대부분은 메인 메모리에서 레지스터로 데이터를 옮겨와 데이터를 처리한 후 그 내용을 다시 레지스터에서 메인메모리로 저장하는 로드-스토어 설계
-

### 캐시

- CPU 내부에 존재
- 컴퓨터 시스템 성능을 향상싴기 위해 주로 cpu 칩 안에 포함되는 빠르고 작고 비싼 메모리
- 메인 메모리에 있는 데이터를 캐시 메모리에 불러와 두고 CPU가 필요한 데이터를 캐시에서 먼저 찾도록 해 시스템 성능 향상

### RAM

- CPU와 별개인 메모리 중 최상위 메모리, 주기억장치이자 최하위 메모리
- 휘발성 기억장치
- SSD,HHD와 Cache의 속도차이를 메꾸기 위해 만들어짐

### Disk (SSD, HDD)

- 비휘발성
